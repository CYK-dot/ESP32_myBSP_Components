# 组件功能
    为已经DIY过的板子提供驱动程序，方便基于此快速展开二次开发。

# 使用说明
    使用时仅需包含单个拓展板头文件，核心板和拓展板的依赖关系由组件内部维护。

# 维护说明

    ## 文件命名
        ESP32型号-主板名称-拓展板名称。

    ## 函数命名
        Board-板子名称-Ctrl/App-下划线-大驼峰功能命名

    ## 函数接口实现
        分为两部分：板载外设接口/业务逻辑接口。

        - 板载外设接口一般不再修改，可用于实现新的业务逻辑。
        - 板载外设接口要确保功能充分暴露。
        - 易错事项：在多次调用的应用场景下，无法让视野完全脱离应用场景，导致esp32center的外设接口抽象出错

        - 业务逻辑即基于该板子实现的功能，开了一个新的项目，就补充一套新的接口，原先接口不做改动。
        - 核心板的业务逻辑接口需要做到板载外设的程度，充分暴露接口，因为核心板本身就是一种模块。
        - 拓展板业务逻辑接口无需确保功能充分暴露，因为代码复用的需求小，如果有复用需求，也可以很快ctrl+cv。
